# 房间体验优化待办

> 范围：仅整理实现细节，不修改现有代码。以下事项面向 WatchTvTogether-Web 前端实现，必要时标注可能需要后端配合的接口能力。

## 1. 房间内添加分享按钮与免登录/带密码跳转支持

- [ ] 在房间页显著位置添加“分享房间”按钮。
  - [ ] 优先放在房间标题、成员/设置区域或播放器上方操作区，避免遮挡播放内容。
  - [ ] 点击后展示分享弹窗或下拉面板，包含可复制链接与复制成功/失败反馈。
- [ ] 设计分享链接格式。
  - [ ] 支持基础链接：`/room/:roomId`。
  - [ ] 支持私有房密码参数，例如 `/room/:roomId?password=...` 或更安全的短期邀请码参数。
  - [ ] 对密码参数进行 URL 编码，避免特殊字符破坏链接。
  - [ ] 明确密码明文放入 URL 的安全风险；如后端支持，优先改为一次性/短期分享 token。
- [ ] 支持未登录用户通过分享链接进入房间。
  - [ ] 进入房间页时检测本地登录态。
  - [ ] 未登录时允许走游客身份或轻量加入流程，不强制跳回登录页。
  - [ ] 如后端当前不支持游客加入，补充后端需求：房间 join/snapshot/token 接口需接受游客身份或临时身份。
- [ ] 支持已登录用户通过分享链接直接进入房间。
  - [ ] 已登录时复用当前用户身份调用加入房间与 snapshot 初始化。
  - [ ] 如果链接带密码，自动将密码传入加入房间/snapshot/Ably token 续签流程。
- [ ] 支持私有房间免再次输入密码。
  - [ ] 从 URL query 读取密码或分享 token。
  - [ ] 加入房间成功后将密码保存在当前页面内存或现有房间状态中，供 snapshot 与 Ably token 续签使用。
  - [ ] 刷新页面时仍可从 URL 重新取值；若用户移除 query，则保持当前既有安全策略。
- [ ] 完善错误处理。
  - [ ] 密码错误、链接过期、房间不存在、权限不足时显示明确提示。
  - [ ] 分享链接中密码缺失且房间为私有时，再展示密码输入框。
- [ ] 验收标准。
  - [ ] 未登录用户打开公开房分享链接可以进入房间。
  - [ ] 已登录用户打开公开房分享链接可以进入房间。
  - [ ] 未登录用户打开带正确密码的私有房分享链接可以进入房间且无需手动输入密码。
  - [ ] 已登录用户打开带正确密码的私有房分享链接可以进入房间且无需手动输入密码。
  - [ ] 错误密码或失效分享参数会展示可理解的错误提示。

## 2. 普通成员不再以只读观看方式加入；控制权由房主通过播放器与队列管理

- [ ] 梳理当前房间角色与权限。
  - [ ] 明确房主/管理员/普通成员各自可见的控件。
  - [ ] 移除“普通成员只读观看”相关文案、状态与 UI 分支。
- [ ] 普通成员进入房间后保持正常观看能力。
  - [ ] 普通成员可以正常播放媒体并接收同步状态。
  - [ ] 普通成员不应拥有影响全房间的控制权限。
- [ ] 房主通过播放器控制全房间进度。
  - [ ] 只有房主/管理员触发的播放、暂停、seek、同步进度事件会调用房间 control 接口并广播。
  - [ ] 普通成员本地操作如被允许存在，应避免向服务端提交全局控制事件。
- [ ] 房主通过队列控制切换视频。
  - [ ] 只有房主/管理员可以新增、删除、排序队列或切换当前视频。
  - [ ] 普通成员界面隐藏或禁用队列编辑入口，并展示“由房主控制队列”的提示。
- [ ] 验收标准。
  - [ ] 普通成员进入房间不会出现“只读观看”模式提示。
  - [ ] 普通成员无法拖动全局进度、暂停全局播放或切换全局视频。
  - [ ] 房主通过播放器操作可以同步影响所有成员。
  - [ ] 房主通过队列切视频可以同步影响所有成员。

## 3. 禁止部分浏览器弹出自带小窗导致用户绕过控制限制

- [ ] 复现并定位浏览器自带小窗类型。
  - [ ] 检查是否为 Picture-in-Picture、Remote Playback、媒体会话通知、系统级媒体浮窗或浏览器原生 video controls。
  - [ ] 分别在 Chrome、Edge、Safari/移动端 WebView 等环境确认表现。
- [ ] 禁用 Picture-in-Picture 入口。
  - [ ] 对 video 元素设置 `disablePictureInPicture`。
  - [ ] 监听 `enterpictureinpicture` 事件，若触发则立即退出并提示当前房间不支持小窗播放。
  - [ ] 对支持相关属性的浏览器设置 `controlsList`，尽量移除 picture-in-picture、download、remote playback 等入口。
- [ ] 禁用远程播放能力。
  - [ ] 对 video 元素设置 `disableRemotePlayback`。
  - [ ] 隐藏或阻止浏览器提供的投屏/远程播放入口。
- [ ] 避免使用会暴露完整原生控制条的配置。
  - [ ] 确认普通成员场景下不渲染完整 `controls`。
  - [ ] 自定义控件时阻止用户通过原生控件拖动进度或暂停全局播放。
- [ ] 补充事件级防护。
  - [ ] 对普通成员本地 seek/pause/play 事件进行权限校验。
  - [ ] 若检测到非房主触发了不允许的进度变更，立即根据房间同步状态回滚到服务端权威进度。
- [ ] 验收标准。
  - [ ] 在支持 Picture-in-Picture 的浏览器中看不到或无法使用小窗入口。
  - [ ] 即使浏览器触发系统媒体控件，普通成员也无法改变全局播放进度或暂停全局播放。
  - [ ] 房主/管理员仍可正常使用房间控制能力。

## 4. 优化视频下方播放/暂停与音量控制，并支持自动隐藏

- [ ] 评估是否可以安全使用原生控制条的子集。
  - [ ] 调研目标浏览器是否支持只显示播放/暂停与音量、隐藏进度条的原生配置。
  - [ ] 如果无法跨浏览器稳定隐藏进度条，则不使用原生完整控制条。
- [ ] 优先方案：重新实现轻量自定义控制条。
  - [ ] 控件仅包含播放/暂停、音量、静音、当前播放状态提示。
  - [ ] 房主/管理员可显示额外进度控制或同步按钮；普通成员不显示可拖动进度条。
  - [ ] 样式与现有房间页视觉统一，支持桌面与移动端布局。
- [ ] 实现自动隐藏逻辑。
  - [ ] 播放中、鼠标静止、触摸无操作一段时间后自动隐藏。
  - [ ] 鼠标移动、触摸屏幕、键盘聚焦、暂停状态、音量面板打开时显示。
  - [ ] 控件获得焦点或使用键盘操作时不得突然隐藏，保证可访问性。
  - [ ] 全屏、非全屏、移动端浏览器、嵌入式 WebView 下都使用同一套状态机控制显示/隐藏。
- [ ] 处理权限与同步。
  - [ ] 房主点击播放/暂停时提交房间 control 并广播。
  - [ ] 普通成员的播放/暂停按钮如保留，应明确是本地音视频行为还是禁用；推荐普通成员只展示状态与音量控制。
  - [ ] 音量与静音仅影响本地，不广播到房间。
- [ ] 验收标准。
  - [ ] 控制条视觉明显优于当前按钮布局。
  - [ ] 控制条在播放中可自动隐藏，并能在鼠标/触摸/键盘操作时恢复。
  - [ ] 普通成员无法通过控制条拖动进度。
  - [ ] 房主可正常播放、暂停和控制同步。
  - [ ] 音量控制在不同浏览器下只影响本地。

## 5. 播放列表名称允许自定义，避免直接展示过长 URL

- [ ] 扩展播放列表条目展示字段。
  - [ ] 为队列条目增加可编辑名称字段，例如 `title`、`displayName` 或 `name`。
  - [ ] 若已有后端字段，优先复用；否则补充后端接口需求。
- [ ] 新增手动 URL 入队时的名称输入。
  - [ ] 添加“显示名称”输入框，可选或必填按产品策略确定。
  - [ ] 用户未填写时，从 URL 自动生成短标题：域名、文件名或截断后的路径。
  - [ ] 对过长名称做长度限制、裁剪与 tooltip 展示。
- [ ] 支持已在队列中的条目改名。
  - [ ] 房主/管理员可编辑播放列表显示名称。
  - [ ] 普通成员只查看名称，不可编辑。
  - [ ] 编辑成功后同步刷新所有成员看到的队列。
- [ ] 保留 URL 可追溯能力。
  - [ ] 列表主展示使用自定义名称。
  - [ ] 需要时可在详情、tooltip 或复制入口中查看原始 URL。
- [ ] 验收标准。
  - [ ] 新增 URL 到播放列表时可以填写自定义名称。
  - [ ] 播放列表优先展示自定义名称，而不是长 URL。
  - [ ] 名称为空时有合理默认值且不会撑破布局。
  - [ ] 房主修改名称后，其他成员能看到更新后的名称。

## 建议的整体实现顺序

- [ ] 先梳理 RoomView 中房间权限、播放器控制、队列管理与视频库入口的现有代码路径。
- [ ] 优先处理安全与权限相关事项：分享链接加入、普通成员控制限制、浏览器小窗绕过问题。
- [ ] 再处理播放控件重构，确保不同角色与不同浏览器行为一致。
- [ ] 最后处理播放列表自定义名称与删除房间内视频库，减少 UI 与数据流冲突。
- [ ] 每个阶段补充对应自动化测试或手动验收记录，尤其覆盖房主、普通成员、未登录访问、私有房密码分享和多浏览器播放行为。
